Terminal Backup Restore Instructions
====================================

This backup contains terminal configuration for Nobara 42 (Fedora-based).

AUTOMATIC INSTALLATION
----------------------
Run the install.sh script to automatically install and configure everything:

    chmod +x install.sh
    ./install.sh

MANUAL RESTORATION
------------------

System-wide Zsh plugins referenced in terminal_backup/.zshrc:

- /usr/share/zsh-autosuggestions/zsh-autosuggestions.zsh
  Restore with: sudo dnf install zsh-autosuggestions

- /usr/share/zsh-syntax-highlighting/zsh-syntax-highlighting.zsh
  Restore with: sudo dnf install zsh-syntax-highlighting

- /usr/share/zsh-autocomplete/zsh-autocomplete.plugin.zsh
  Restore with: sudo dnf install zsh-autocomplete
  (or clone from https://github.com/marlonrichert/zsh-autocomplete)

Local plugin locations checked (no directories found to copy):
- ~/.oh-my-zsh/custom/plugins
- ~/.local/share/zsh/plugins

GIT CONFIGURATION
-----------------

Global Git configuration is backed up in .gitconfig (user: haint, email: thanhhai9x98@gmail.com).
The install.sh script will automatically restore this file to ~/.gitconfig, backing up any existing one.

To manually restore:
    cp .gitconfig ~/.gitconfig

VSCODE RESTORATION
------------------

VSCode settings are backed up in the vscode/ directory:

- vscode/settings.json     -> ~/.config/Code/User/settings.json
- vscode/keybindings.json  -> ~/.config/Code/User/keybindings.json (if exists)
- vscode/snippets/         -> ~/.config/Code/User/snippets/ (if exists)
- vscode/extensions.txt    -> List of installed extensions

To manually restore VSCode:

1. Install VSCode:
   sudo rpm --import https://packages.microsoft.com/keys/microsoft.asc
   sudo sh -c 'echo -e "[code]\nname=Visual Studio Code\nbaseurl=https://packages.microsoft.com/yumrepos/vscode\nenabled=1\ngpgcheck=1\ngpgkey=https://packages.microsoft.com/keys/microsoft.asc" > /etc/yum.repos.d/vscode.repo'
   sudo dnf install -y code

2. Restore settings:
   mkdir -p ~/.config/Code/User
   cp vscode/settings.json ~/.config/Code/User/

3. Install extensions:
   while read ext; do code --install-extension "$ext"; done < vscode/extensions.txt

KILO CODE EXTENSION
-------------------

Kilo Code configuration is backed up in:
- vscode/globalStorage/kilocode.kilo-code/ -> ~/.config/Code/User/globalStorage/kilocode.kilo-code/

This includes Kilo Code's modes, settings, and cached data.

The settings.json also contains kilo-code.allowedCommands and kilo-code.deniedCommands settings.

To manually restore Kilo Code configuration:
    mkdir -p ~/.config/Code/User/globalStorage
    cp -r vscode/globalStorage/kilocode.kilo-code ~/.config/Code/User/globalStorage/

QDRANT VECTOR DATABASE
----------------------

The install.sh script automatically sets up Qdrant using Podman with systemd for auto-start on boot.

What gets configured:
- Qdrant storage directory: ~/qdrant_storage
- Systemd user service: ~/.config/systemd/user/qdrant.service
- Ports exposed: 6333 (HTTP API), 6334 (gRPC)
- User lingering enabled for boot startup without login

To manually set up Qdrant:

1. Install Podman:
   sudo dnf install -y podman

2. Create storage directory:
   mkdir -p ~/qdrant_storage

3. Pull the Qdrant image:
   podman pull qdrant/qdrant

4. Create the systemd service file at ~/.config/systemd/user/qdrant.service:
   
   [Unit]
   Description=Qdrant Vector Database Container
   After=network-online.target
   Wants=network-online.target

   [Service]
   Type=simple
   Restart=always
   RestartSec=5
   ExecStartPre=-/usr/bin/podman stop -t 10 qdrant
   ExecStartPre=-/usr/bin/podman rm qdrant
   ExecStart=/usr/bin/podman run --rm --name qdrant \
       -p 6333:6333 -p 6334:6334 \
       -v $HOME/qdrant_storage:/qdrant/storage:Z \
       qdrant/qdrant
   ExecStop=/usr/bin/podman stop -t 10 qdrant

   [Install]
   WantedBy=default.target

5. Enable and start the service:
   systemctl --user daemon-reload
   systemctl --user enable --now qdrant

6. Enable lingering for boot startup:
   loginctl enable-linger $USER

Managing Qdrant:
- Check status: systemctl --user status qdrant
- Stop: systemctl --user stop qdrant
- Start: systemctl --user start qdrant
- View logs: journalctl --user -u qdrant -f
- Access API: http://localhost:6333
- Web UI: http://localhost:6333/dashboard

GODOT ENGINE
------------

Godot Engine configuration is backed up in the godot/ directory:

- godot/editor_settings-4.5.tres -> ~/.config/godot/editor_settings-4.5.tres
- godot/feature_profiles/       -> ~/.config/godot/feature_profiles/
- godot/script_templates/       -> ~/.config/godot/script_templates/
- godot/text_editor_themes/     -> ~/.config/godot/text_editor_themes/

VS Code Integration (preserved in editor_settings):
- External editor: /usr/bin/code
- Flags: {project} --goto {file}:{line}:{col}
- Use external editor: enabled

Installation:
1. Run the install.sh script - it will ask if you want to install Godot
   OR run the dedicated script:
   bash terminal_backup/install_godot.sh

To install a specific version:
   GODOT_VERSION=4.3 bash terminal_backup/install_godot.sh

The installer will:
- Download Godot from GitHub releases
- Install to ~/.local/bin/godot
- Restore configuration from backup (including VS Code integration)
- Create desktop entry for application menu

To manually restore config only:
   mkdir -p ~/.config/godot
   cp -r godot/* ~/.config/godot/

To uninstall:
   bash terminal_backup/install_godot.sh --uninstall

CISCO PACKET TRACER
-------------------

The Cisco Packet Tracer installer (CiscoPacketTracer_Ubuntu_64bit.deb) is now
included in this backup and will be automatically detected during installation.

Installation:
1. Run the install.sh script - it will ask if you want to install Packet Tracer
   OR run the dedicated script:
   bash terminal_backup/install_packettracer.sh

The installer will:
- Automatically detect the bundled .deb file in the backup directory
- Fall back to searching $HOME if not found locally
- Install required Qt5 dependencies
- Extract and convert the .deb package for Fedora/Nobara
- Configure desktop integration
- Add --no-sandbox flag for compatibility
- Restore the custom desktop shortcut (cisco-pt.desktop) to ~/.local/share/applications/

Desktop Shortcut:
The backup includes cisco-pt.desktop, which is the application menu shortcut.
It will be automatically restored during installation to ~/.local/share/applications/.

To get a newer version:
1. Download from Cisco NetAcad or SkillsForAll:
   - https://www.netacad.com/portal/resources/packet-tracer
   - https://skillsforall.com/resources/lab-downloads
2. Replace the .deb file in terminal_backup/ or place in $HOME

To uninstall:
    bash terminal_backup/install_packettracer.sh --uninstall

ADDITIONAL APPLICATIONS
-----------------------

The install.sh script can optionally install additional applications:
- Google Chrome (via DNF with google-chrome repo)
- Dropbox (via DNF with Dropbox repo)
- Discord (via Flatpak from Flathub)
- Obsidian (via Flatpak from Flathub)
- Anki (via Flatpak from Flathub)

To install manually:

1. Google Chrome:
   sudo dnf install -y fedora-workstation-repositories
   sudo dnf config-manager setopt google-chrome.enabled=1
   sudo dnf install -y google-chrome-stable

2. Dropbox:
   sudo dnf config-manager addrepo --from-repofile=https://linux.dropbox.com/fedora/dropbox.repo
   sudo dnf install -y dropbox nautilus-dropbox

3. Flatpak apps (Discord, Obsidian, Anki):
   flatpak remote-add --if-not-exists flathub https://flathub.org/repo/flathub.flatpakrepo
   flatpak install -y flathub com.discordapp.Discord
   flatpak install -y flathub md.obsidian.Obsidian
   flatpak install -y flathub net.ankiweb.Anki

Or run the dedicated script:
   bash terminal_backup/install_apps.sh
